Ajax简介、优劣势、应用场景

Ajax简介
  • Asynchronous Javascript And XML 的缩写，即异步的javascript 和 XML，主要利用XmlHttpRequest对象来与服务器进行异步通信
  • Ajax 是一种用于创建快速动态网页的技术。通过在后台与服务器进行少量数据交换，可以使网页实现异步更新，即可以在不重新加载整个网页的情况下，对网页的某部分进行更新
优势
  • 不用重新加载整个页面就可以实现局部网页的更新，用户体验更好
  • 由于是异步，可以更快响应
  • 减少了请求，减轻服务器的负担
劣势
  • 对搜索引擎支持性不高
  • 无法后退
  • 不利于SEO
  • 无法用URL直接访问
应用场景
  1. 数据验证（表单验证）
  2. 按需取数据
  3. 自动更新页面（实时更新评论）

创建Ajax的步骤

原理：
通过XMLHttpRequest对象来向服务器发异步请求，从服务器获得数据，然后用javascript来操作DOM而更新页面。这其中最关键的一步就是从服务器获得请求数据

一个完整的Ajax请求，最少需要4个参数

  1. 请求方式（get或post）
  2. url
  3. 字符串类型的参数data（参数一般为对象，先将其拼接为字符串）
  4. 回调函数
1.创建XMLHttpRequest对象

新版本的浏览器
var xhr = new XMLHttpRequest( );
老版本的浏览器
var xhr = new ActiveXObject("Microsoft.XMLHTTP");

3.发送请求体

发送请求为xhr.send( )，但是根据请求方式的不同，会有不同的处理结果

1.”GET”
get请求方法的方式是将要提交的参数直接写到open方法的url参数中，此时send方法的参数为null或空

2.”POST”
post请求方式要使用请求头setRequestHeader( )来添加HTTP头，然后再往send方法中添加参数

4.处理响应

监听响应，判断响应是否成功。如果成功就获取响应头的Content-Type类型，判断如何进行解析

readyState是Ajax的处理过程，监听不同的readyState的值表示不同的进程

  • 0：请求尚未初始化，即还没有调用请求行open( )
  • 1：请求已经建立，但是还没有发送，即没有调用请求体send( )
  • 2：请求已发送，正在处理中（通常现在可以从响应中获取内容头）
  • 3：请求在处理中，通常响应中已有部分数据可用了，但是服务器还没有完成响应的生成
  • 4：响应已完成

关于status是HTTP状态码，一般只有状态码为200才表示请求成功

请求成功后会根据响应头判断所返回的数据
xhr.getResponseHeader("Content-Type");
数据有两种格式
1.responseText JSON字符串形式的响应数据
2.responseXMLXML形式的响应数据

一般将返回的数据转换为javascript对象的形式

JSON字符串转换为对象
result=JSON.parse(xhr.responseText);
